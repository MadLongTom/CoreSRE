## 1. 业务背景

大型企业在多个业务领域中已部署大量 AI 智能体（Agent），但这些智能体多数以孤立方式运行——无法共享上下文、无法协同完成跨领域复杂任务。同时，企业内部的异构系统（API、MCP 工具、数据源）缺乏统一接入层，导致每次集成新智能体或新工具都需要大量重复性工作。

A2A（Agent-to-Agent）协议的标准化，使跨平台、跨供应商的智能体通信成为可能。但企业需要一个**统一编排层**来管理这些智能体的注册、发现、调度与协同，而不是让每个智能体自行处理互联问题。

## 2. 业务目标

构建 **CoreSRE——分布式智能体编排与协同平台**，作为企业 AI 运营体系的上游编排层：

| # | 目标 | 可量化指标 |
|---|------|-----------|
| BG-1 | 统一管理企业内多类型智能体的注册、发现与生命周期 | 支持 ≥50 个并发注册智能体，覆盖 A2AAgent + ChatClientAgent + WorkflowAgent 三种类型 |
| BG-2 | 提供标准化的 A2A 协议通信能力 | 100% 符合 A2A 协议规范 |
| BG-3 | 统一接入异构外部 API/MCP 工具 | 支持 REST API + MCP 两种接入协议 |
| BG-4 | 可视化工作流编排多智能体协同，工作流自身可作为 Agent 复用 | 用户可通过拖拽完成工作流构建；支持 Sequential/Concurrent/Handoff/GroupChat 四种编排模式 |
| BG-5 | 至少完成 1 个 AIOps 自动化运维闭环场景 | 端到端告警→分析→处置闭环 |
| BG-6 | 全链路可观测性 | Traces/Metrics/Logs 覆盖所有智能体调用 |
| BG-7 | 保障自动化执行的安全性与可控性 | 身份认证、人工审批、权限隔离、审计日志 |

## 3. 业务问题域

基于课题要求，CoreSRE 需解决以下六大业务问题：

### 3.1 异构 API/MCP 统一接入与管理

**问题**: 企业内各系统提供的 API 协议、认证方式、数据格式各不相同，每次新增工具都需定制开发适配器。

**期望**: 提供统一网关，支持 REST API 和 MCP 协议自动适配；统一凭据托管；内置配额管理与熔断策略。

### 3.2 API 文档到平台节点的自动生成

**问题**: 接入新 API 后，还需要手动在平台中创建对应的工具节点，效率低且容易出错。

**期望**: 上传 OpenAPI/Swagger 文档后，自动解析并生成平台可用的工具节点（MCP Tool 或 A2A AgentCard）。

### 3.3 Agent 事件驱动与策略化工具调用

**问题**: 智能体调用外部工具时缺乏统一的事件驱动机制和调用策略（重试、超时、回退）。

**期望**: Agent 可基于事件（告警、定时、消息）自动触发工具调用；支持短时工具（一次性调用）和长时工具（持续守护任务）的统一编排。

### 3.4 大规模数据实时分析与智能决策

**问题**: 日志、指标等运维数据量大且分散，人工分析效率低，智能体需要能实时获取并分析这些数据。

**期望**: 接入结构化与半结构化数据源（日志、Metrics、Traces）；提供统一语义层供智能体查询推理。

### 3.5 自动化任务执行安全与可控

**问题**: 智能体自动执行操作（如重启服务、修改配置）存在安全风险，需人工确认与审计。

**期望**: 平台需内置身份认证（JWT）以识别操作者身份；关键操作需人工审批（Human-in-the-loop）；完整操作审计日志；基于角色的权限隔离（RBAC）；异常自动熔断。

### 3.6 多 Agent 可视化监控

**问题**: 多智能体并行工作时，运维人员无法直观了解各智能体状态、工作进度和交互关系。

**期望**: 提供全景可视化面板：智能体拓扑图、任务状态流、消息链路追踪、实时日志聚合。

## 4. 利益相关者

| 角色 | 需求关注点 |
|------|-----------|
| **平台管理员** | 智能体注册/管理、工具接入、权限配置、系统监控 |

| **运维工程师 (SRE)** | AIOps 场景、告警处理、根因分析、自动修复 |

## 5. 业务约束

| 约束 | 说明 |
|------|------|
| 技术栈 | 后端 .NET 10 + ASP.NET Core，前端 React |
| 协议标准 | A2A 协议 + MCP 协议 |
| 框架 | Microsoft Agent Framework |
| 数据库 | PostgreSQL（通过 EF Core + Npgsql；Aspire 编排 PostgreSQL 容器） |
| 会话持久化 | AgentSession 基于 PostgreSQL 实现自定义 `AgentSessionStore`（框架仅提供 InMemory 实现） |
| 云原生 | .NET Aspire + Kubernetes |
| 可观测性 | OpenTelemetry (Traces + Metrics + Logs) |
| AI 能力 | OpenAI Compatible API |
| 部署环境 | 本地开发机（可扩展至 K8s 集群） |

## 7. 项目范围（高层级）

### 在范围内 (In Scope)

- 智能体注册中心（Agent Registry）
- A2A 协议通信层
- MCP/API 工具统一接入网关
- 可视化工作流编排器（前端 React）
- AIOps 运维场景（告警联动 / 根因分析）
- OpenTelemetry 全链路可观测性
- 人工审批与安全管控
- .NET Aspire 编排与 Dashboard

### 不在范围内 (Out of Scope)

- 多租户 SaaS 化
- 生产级 Kubernetes 部署（仅提供 Aspire manifest，不含运维）
- 自定义 LLM 训练 / 微调
- 移动端 App
- 非 OpenAI Compatible 的模型适配（如本地 Ollama 可选但非必须）
